# 正規言語は受理される

オートマトンは Q × Σ に対して関数 δ を決めることで定まる。入出力のふるまいが同じ状態およびシステムは等価であり、そのことはグラフの同型性などで確かめられる。ここで特に FAM が受理するような言語 L(M) は正規言語と言われる。言語とは集合に他ならず、そもそも正規言語はオートマトンというシステムとは別に、自然言語やプログラミング言語を"意味抜き"し、集合演算 ∪, ・, ＊ で構成されるものである。このように抽象化された言語は形式言語とよばれ、その言語記述能力によって L3 ⊆ L2 ⊆ L1 ⊆ L0 と階層的に分類される。前述の正規言語は L3 のことであり、認識のためのカウンタを必要としない言語である。

# 形式言語は生成される

形式言語は、生成や構成や受理により定義される。

# 言語は無限の記述を目指して拡張される

SL(単純言語)は接頭辞性質を有し、SDG および simpleDPDA で記述される。SL ⊂ DL であるような DL(決定言語)は DPDA で受理されるような言語である。DL ⊂ CFL であるような CFL(文脈自由言語)は自己埋め込み性を有し、CFG および NPDA で記述される。このように＊を記述する方向に言語は拡張されていくが、なおも CFG は 3 箇所以上の部分の無限個数の制御はできない。ちなみに、構文解析して W がどの L に所属するかを考えるのが所属問題である。

# 問題解決は TM で写し取れる

PSL は PSG で生成され、かつ、TM で受理される。PSG とは、生成規則が α → β であるような文法であり、TM は M = ( Q, Γ, Σ, δ, q0, b, F ) であるようなシステムである。PSL は機能的可算、すなわち、枚挙ができ、機能的な言語のクラスを真に含む。帰納的な言語というのは CSG、文脈依存文法のことである。ここで TM に着目すると、興味深い性質が 2 つある。一つは模倣性で、 TM は変形 TM を模倣できるのみならず、UTM は任意の他の TM を模倣し、これはプログラム内蔵型で実装できる。もう一つは計算可能性で、TM は計算する関数であり、とくに、必ずいつか停止する TM の表す関数は計算可能であるという。また、停止する TM の動作で記述できる手続きをアルゴリズムといい、ある問題の YES-NO を TM の受理の当否に対応させられるならその問題は決定可能であるという。
